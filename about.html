About This Project

Handheld Heritage — 交互式 3D 复古掌机博物馆

University of Sussex · 3D Apps · A2 Assignment · 2026

设计理念

本项目以"掌机演化"为主线，聚焦 Game Boy（1989）、GBA SP（2003）、PSP（2004）三代里程碑产品。 选题原因有三：其一，三款机型在硬件材质上截然不同（哑光灰塑料 → 金属漆翻盖 → 钢琴黑亮面）， 天然适合展示 PBR 材质的表现力；其二，三代之间存在清晰的技术演进叙事，便于构建博物馆式的叙事框架； 其三，作为游戏文化符号，掌机具有强烈的情感共鸣，能吸引观众深度交互。

UI 采用深色博物馆风格，以金色点缀为主，模仿实体博物馆展柜的沉浸感。 左侧为控制面板，右侧为全屏 3D 展示区，减少界面干扰，让模型成为视觉焦点。

技术决策

Three.js
业界主流 WebGL 库，提供完整的 PBR 材质系统、GLTFLoader、EffectComposer 后期处理链，无需自行管理 WebGL 上下文。
Vite
原生 ES module 支持，热模块替换（HMR）让 shader 迭代极快；构建产物体积小，适合作业提交。
Bootstrap 5
评分点要求，提供响应式网格和组件基础，自定义 CSS 变量实现深色主题覆盖。
GSAP
专业动画引擎，处理相机位置插值、翻盖旋转、UI 过渡比原生 CSS transition 更精确可控。
自定义 GLSL Shader
CRT 屏幕 shader 实现扫描线、RGB 通道偏移、桶形畸变，赋予屏幕复古质感，满足 Deeper Understanding 评分项。
UnrealBloomPass
后期处理发光效果，让掌机屏幕和 LED 指示灯具备真实的光晕，提升画面质量。
RGBELoader + HDRi
高动态范围环境贴图，使 PSP 钢琴黑表面能真实反射周围环境，体现 PBR 材质的金属感。

MVC 架构

  • Model（src/js/model/consoles.js）：存储三台掌机的元数据，与渲染逻辑完全解耦。
  • View（src/js/view/）：Three.js 场景初始化、GLTF 加载、后期处理、GSAP 动画封装。
  • Controller（src/js/controller/）：DOM 事件绑定、用户输入分发、音频控制，不直接操作 Three.js 对象。

可访问性

  • 所有交互按钮含 aria-label 或可见文字，屏幕阅读器可读。
  • Toggle 按钮使用 aria-pressed 状态，状态变化时同步更新属性。
  • 3D 画布附有描述性 aria-label，加载进度区域有 aria-live="polite"。
  • 键盘导航：Tab 键遍历所有控制按钮，Enter / Space 触发交互；方向键可旋转模型（通过 OrbitControls 键盘绑定）。
  • 颜色对比度：所有文字与背景的对比度 ≥ 4.5:1，符合 WCAG 2.1 AA 标准。
  • 运动减弱：检测 prefers-reduced-motion，开启时禁用 GSAP 过渡动画。

已知问题 & 修复记录

  • Safari 不允许无手势自动播放视频 → 所有 <video> 添加 muted 属性，点击"开机"后再取消静音。
  • 低端 GPU 上 Bloom 后期处理帧率下降 → 增加性能检测，低于 30fps 时自动降低 Bloom 强度。
